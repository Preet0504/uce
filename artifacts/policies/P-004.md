ID: P-004
Title: Agent Configuration Integrity and Credential Isolation Policy
Type: Internal / AI Safety
Description: AI agent records must retain their identity and configuration fields intact so that agent behaviour is deterministic and auditable. OAuth credentials stored in the `accounts` table and associated with agents must be isolated from read access by any service other than the credential-vault service. Changes to `instructions`, `access_token`, or `refresh_token` must be logged and require elevated permissions.
Controls:
  - C1: `agents.name` and `agents.instructions` are immutable once the agent is activated; updates require a version increment and create a new immutable snapshot.
  - C2: `accounts.access_token` and `accounts.refresh_token` are readable only by the `credential_vault` service role; no other service has column-level SELECT on these fields.
  - C3: Any UPDATE to `accounts.access_token` or `accounts.refresh_token` triggers an audit event captured by the security event stream.
  - C4: Agents MUST NOT log or expose their own `access_token` or `refresh_token` values in responses, tool outputs, or error messages.
  - C5: `meetings.name` is validated as non-null, max-length 255, and sanitised to prevent stored XSS or SQL injection via meeting names in dynamic queries.
Enforces: RQ-004, RQ-007, RQ-010
Enforcement Mode: block
GDPR Relevance: Agent `instructions` may contain implicit personal data (names, addresses used as examples). OAuth tokens in `accounts` are high-sensitivity personal data (Art. 4(1)). Column-level access control is an Art. 32 technical measure. Token revocation on deletion is an Art. 17 erasure obligation.
